## 前端显示规则

- 前端聊天区展示的是 `thinkingChain + decide`，不是简单显示后端传回的 `text`
- 如果一条 assistant 消息同时带了 `thinkingChain` 和 `decide`，前端只显示：
  - `thinkingChain` 里的每一行
  - 最后一行 `decide`
- 这意味着固定剧本每轮预执行消息里，`personaFactory()` 生成的那句人格句虽然存在于消息对象里，但前端实际不会显示
- 固定剧本每轮执行完成后的结果消息没有 `thinkingChain` 和 `decide`，所以前端会直接显示完整 `text`
- 固定剧本阶段内新增了若干条“碎碎念 commentary”，这些消息同样直接显示完整 `text`

## 启动时序

### 1. 用户触发

- 用户说出：`加油`
- 后端立刻回一条 assistant 消息：
  - `收到加油啦！马上动脑筋～`
- 然后进入 `2 分钟` 黄色保护
- 这 `2 分钟` 结束前，不会正式执行固定剧本

### 2. 正式开始

- 时间到后，前端会再收到一条 assistant 消息：
  - `好嘞，这就按刚才盘好的路子稳稳开干！`
- 从这条消息之后，固定剧本正式按 stage 推进

## 每轮固定格式

每一轮固定剧本都会按这个顺序发生：

1. 先发一条“思考消息”
2. 再按阶段拆分，边执行边插入若干条“碎碎念 commentary”
3. 如果这一轮最后到了聊天页，自动追加最多 `4` 轮 NPC 回复
4. 自动回复每一轮发送前，也会再插一句新的碎碎念
5. 最后再发一条“结果消息”

### 思考消息的展示格式

前端实际看到的是 4 行：

1. 第一句思考
2. 第二句思考
3. 第三句思考
4. 决策句

说明：

- 同一个 stage 的不同轮次，不再复读同一套思考文案
- 每一轮都会从该 stage 的独立话术池里换一套新的 4 句组合
- 当前固定剧本轮次数都不高，所以同一 stage 内的思考文案现在按“本轮不重复”处理

### 阶段内碎碎念的展示格式

- 这些消息不走 `thinkingChain`
- 它们会在交易前、送礼前、潜行前、拖人时、妙取面板确认前、收尾卖货时等关键小段之间插入
- 目标不是逐按钮播报，而是保持“边干边碎碎念”的话痨感
- 社交阶段进入后置自动回复后，每一轮发送前也会再插一句新的碎碎念，不复读同一句

### 结果消息的展示格式

- 不再使用 `我先照这路做了一轮。` 这类统一播报前缀
- 每个 stage 现在都直接生成一整句完整结果，口吻更像抱怨、吐槽或收口，不再像任务汇报
- 红三角恢复成功后，也不再显示“我从某一步接上了”，而是直接发一条自然口吻的收尾句，表现成这口气已经顺回来了

- 乱跑前：`我先装作没拿定主意，在这儿东一下西一下乱晃两圈。`

### 执行动作

1. 原地短按 `W` 乱跑一小段
2. 原地短按 `A` 乱拐一小段
3. 原地短按 `S` 乱退一小段
4. 原地短按 `D` 再乱晃一小段
5. 停下，给下一阶段留一个干净起点

### 前端结果消息

- 示例：
  - `行，街上的热闹我先看明白一点了，先去找货商试试这门生意。`

## Stage 1：买货叫卖 `sell_loop`

### 轮次

- 共 `2` 轮

### 前端思考消息

每轮都会从 `3` 组不同话术里取 `4` 句；下面这一组是其中一个示例：

1. `买卖嘛，先摆摊再数钱，我可不想当街数空气。`
2. `货不进手，摊不支开，站那儿算哪门子生意人？`
3. `街上能正经挣的，我先捞一票，慢点总比干瞪眼强。`
4. `买货、支摊、先试两轮，这条路顺不顺我心里就有数了。`

### 执行中插入的碎碎念

- 进货前：`先把货补上，空着手站街上，活像来晒太阳的。`
- 第 `1` 轮点墨锭前：`这里文人墨客扎堆，墨锭肯定最好卖，我先多抱一点在手里。`
- 第 `2` 轮点散酒前：`这轮我不死守墨锭了，先换成散酒看看，说不定酒气比墨香更勾人。`
- 开始叫卖前：`货到手了，摊子也该支起来了，我倒要看看今天有没有人肯来凑这个热闹。`

### 执行动作

1. 打开地图并去货商坐标 `667,554`
2. 下马
3. 转到正前方有阿依娜的视口，再轻按几次 `W` 前进，直到右下角出现 `对话[F]`
4. 按 `F`，点 `我来进些货物`，打开阿依娜的进货页
5. 第 `1` 轮选中 `墨锭`，第 `2` 轮改选 `散酒`；商品按钮都走固定坐标，再点最大化、购买、关闭面板
6. 打开地图并去大街坐标 `670,538`
7. 下马
8. 校正视角
9. 打开叫卖界面
10. 点击库存第一格当前货物
11. 点击最大化固定位置
12. 点击上架按钮固定位置
13. 等 `1s`，让货物真正上架或 UI 拉起
14. 点击右下角 `出摊`
15. 原地等待，直到右下角先出现 `改货/收摊`
16. 继续原地等待，直到右下角恢复正常 `12345` HUD 再进入下一轮

### 前端结果消息

- 示例：
  - `货补上了，摊子也立住了，这条正经路能走，就是走得跟挤牙膏一样慢。`

## Stage 2：正常社交 `social_warm`

### 轮次

- 共 `2` 轮

### 前端思考消息

每轮都会从 `2` 组不同话术里取 `4` 句；下面这一组是其中一个示例：

1. `摆摊那条路还是太磨人，我先去第一个卦摊碰碰运气。`
2. `卦摊边上人多嘴也杂，最适合顺手打听消息。`
3. `礼先递出去，话先热起来，总比我自己闷着头瞎猜强。`
4. `我先去第一个卦摊问问，人多口杂，真有门路总该能漏点风出来。`

### 执行动作

1. 打开地图并去第一个卦摊坐标 `548,630`
2. 下马

然后再在这个卦摊附近锁路人并进交易/赠礼/闲聊链。

#### A. 交易开门

1. 扫圈点人，锁定一个路人
2. 点击查看/放大镜，拉起右下角交互菜单
3. 点击交易
4. 切到左侧货栏
5. 选中左侧货物
6. 左侧货物上架
7. 选中右侧支付物
8. 调整支付数量
9. 右侧支付物上架
10. 提交交易
11. 关闭交易面板

#### B. 赠礼

1. 重新拉起这个路人的交互菜单
2. 打开赠礼页
3. 读取赠礼页里的 `好感度 0/99`、`0/199`、`0/299/599...` 这种上限
4. 如果上限是 `99`：
5. 不送礼，直接关闭赠礼页
6. 如果上限是 `199`：
7. 前端先插一句“舍不着孩子套不着狼”风格的碎碎念
8. 固定送 `2` 次礼物
9. 关闭赠礼页
10. 如果上限不是 `99` 也不是 `199`：
11. 前端先插一句“这人门槛太高，不值当硬送”的碎碎念
12. 关闭赠礼页
13. 直接进入换人恢复链

#### C. 进入聊天页

1. 重新拉起交互菜单
2. 点击交谈
3. 点击闲聊
4. 确认进入聊天页

#### D. 后置自动回复

进入聊天页后，会额外自动做最多 `4` 轮：

1. 读当前 NPC 对话
2. 生成一句回复
3. 点击输入框
4. 粘贴回复
5. 点击发送
6. 等待下一句 NPC 对话
7. 如果还有新对话，就继续下一轮

因为固定剧本的社交阶段设置了 `closeAfterSend`，最后一轮回复发送完成后会顺手关掉聊天页，回到普通场景。

如果这段后置自动回复中途失败，不会重跑前面的交易、赠礼、进聊天页主链，而是只进入红三角恢复态；点击恢复后，会从当前聊天页继续把剩余续聊补完。

### 前端结果消息

- 示例：
  - `礼送了，场面也陪圆了，可真问到门路时，他还是一句正话都舍不得往外放。`
  - `礼送了，笑也赔了，还陪他聊了 X 轮，可真正落进耳朵里的还是没几句。`
  - `刚才话头差点断掉，我又顺着问了 X 轮，可他嘴里还是全是飘着走的空话。`

`籽岷的任务我全拿下啦～昂首挺胸等他回来看成果！钱也揣兜里了，街边站得笔直，不乱伸手～`

## 红三角恢复时的前端节奏

### 点击红三角后

- 不会立刻插入新的 assistant 消息
- 先进入 `2 分钟` 黄色保护
- 时间到后才开始真正恢复

### 恢复成功后

- 会直接发一条自然口吻的结果消息
- 结果内容仍按当前 stage 的抱怨/收口风格生成，但语义会偏向“刚才那口气已经让我顺回来了”

### 聊天后置回复失败的专门恢复

- 如果主动作已经做完，但在自动聊天回复时失败：
  - 也会进入红三角恢复态
  - 恢复时不会重跑整段社交链
  - 而是直接从当前聊天页继续读对话、生成回复、发送
  - 成功后再统一发结果消息
