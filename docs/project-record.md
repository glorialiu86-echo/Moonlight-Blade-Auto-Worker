## 本轮新增结论

- 固定剧本自动化现在是唯一主流程真源，LLM 只负责前端思考外观与局部聊天内容
- 路人 NPC 的唯一稳定交互真源已收敛为阶段化点击链
- 后端只有在真正进入聊天界面后，才允许继续尝试自动发闲聊回复
- 已新增 `docs/specs/town-npc-gift-and-chat-flow.md`，把聊天入口、交易拿礼物、赠礼提好感三条链路合并成统一真源
- 已新增 `docs/specs/fixed-script-automation-flow.md`，明确固定剧本自动化与前端展示边界
- 已确认 NPC 分类规则：蓝名普通 NPC 走 `10` 点阈值，蓝名下带白色 `<>` 称号的特殊 NPC 走 `50` 点阈值
- 已新增 `docs/specs/npc-atomic-action-spec.md`，把 NPC 社交执行链拆成独立原子 action 规格
- 已补出可单独验证的底层原子动作：`短距离前进`、`右键拖拽转视角`、`相对坐标点击`
- `走动` / `转视角` 的主判定已收敛为本地规则视觉：先采样待机噪声基线，再用场景差分做 `pass/review/fail`
- `review` 边界样本已自动落盘到 `data/motion-review/`，并由本地文本模型异步做低频复核；模型不参与主控制判定
- 路人 NPC 交互已开始改成快速突击链：优先点右侧街道人群，命中目标后立即推进 `查看 -> 交谈 -> 闲聊`，不再在前半段频繁 OCR 和乱动镜头
- 地图寻路 owner 已统一到 `travel_to_coordinate`，由它负责开图、填坐标、前往、关图、小地图坐标校验和 reroute
- `Tab` 现在只负责社交换人，不再承担首次选人和拉起右下角固定 UI
- `妙取` 已从闷棍链里彻底拆出，改成独立潜行妙取链，不再跟在闷棍后面硬接
- 失败时不再向前端吐失败台词，唯一可见恢复入口就是红色三角按钮

## 当前主要风险

- 现在已经有固定恢复链，但没有完成 Windows 真机上的长时稳定性回归，因此失败密度和恢复命中率仍待验证
- 当前货商购买链路和叫卖链路还依赖旧点位/未完整重标，Windows 真机上仍可能出现 sale 段细节偏移
- 新的原子 action 拆分还没在 Windows 真机上逐步验证，因此当前风险主要在真实界面切换细节
- 真实输入已经接入，后续所有主动动作都需要继续收紧安全边界
- 当前本地模型复核仍基于规则指标文本而不是视觉图像语义，本轮先完成“边界样本留痕 + 低频复核器”闭环
