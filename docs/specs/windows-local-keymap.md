# Windows Local Keymap
状态：执行完成

## 目的

记录当前这台 Windows 机器上，《天涯明月刀手游》自动化实现所依赖的实际键位和交互结论，避免后续把公开资料、旧默认键位和本机真实行为混在一起。

## 真源

- 本机键位设置截图
- 本轮用户口头确认

后续如果游戏内键位被改动，应直接更新这份文档，不再以网页资料作为唯一依据。

## 当前确认的键位

- `W`: 前进
- `A`: 左移
- `S`: 后退
- `D`: 右移
- `Shift`: 翻滚
- `Space`: 三段跳
- `F`: 交互
- `Esc`: 菜单
- `M`: 地图
- `N`: 背包

## 当前确认的交互结论

### 路人 NPC

- 城镇里的路人 NPC 不能把 `F` 当成稳定主链路。
- 当前本机已确认的稳定链路是点击链，不是键盘交互链。
- 唯一主链路固定为：
  - 点击 NPC
  - 点击 NPC 旁边的 `查看`
  - 点击右下角 `交谈`
  - 点击 `交谈` 面板里的 `闲聊`
  - 如果出现 `闲聊确认`，点击确认
  - 如果没出现确认，而是直接进入聊天界面，则直接视为成功进入聊天
- 只要目标是街上游荡的路人 NPC，后续自动化都应以这条点击链为准。

### 功能型 NPC

- 功能型 NPC 例如货商、商店入口、明确带功能面板的对象，可以保留 `F` 作为候选交互键。
- 但只有在识别到功能性交互态时，才允许尝试 `F`。
- 不允许把 `F` 同时作为路人 NPC 和功能型 NPC 的统一 owner。

## 当前自动化 owner 划分

- `路人 NPC 对话`: 点击链路
- `功能型 NPC 入口`: 条件成立时可尝试 `F`
- `持续跑动/逼近目标`: 本地高频循环
- `策略规划`: 文本模型

## 保持不变的点

- 本机 `F` 键位绑定依然存在，没有否认这个绑定。
- 变化的是自动化控制策略，不是游戏设置本身。
- 当前文档只覆盖已经确认的键位和交互，不扩展到尚未验证的复杂战斗键位。
