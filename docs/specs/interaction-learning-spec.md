# 交互学习记录规格
状态：执行完成

## 目的

为《天涯明月刀手游》本地自动化提供一套低风险的“自我总结”机制。

这里的“学习”不是让运行中的 Agent 自己改代码、自己改控制链路，而是：

- 自动记录每次 NPC 交互的事实样本
- 自动统计成功率、点击点表现、失败原因
- 自动生成可读总结
- 再由工程实现把稳定经验收敛为规则升级

## 当前原则

1. 运行时只允许记录、统计、总结，不允许直接改控制 owner。
2. 路人 NPC 对话仍由点击链路控制，不能因为一次总结就切回 `F`。
3. 所有失败都必须可追踪，不能 silent failure。
4. 规则升级由仓库代码完成，不由运行中的模型直接落盘覆盖主逻辑。

## 当前落盘内容

- 样本日志：`data/interaction-learning/npc-interaction-samples.jsonl`
- 最新统计：`data/interaction-learning/latest-summary.json`
- 最新可读报告：`data/interaction-learning/latest-summary.md`

## 当前记录字段

- 指令内容
- 来源：用户或自主回合
- 场景
- 所选策略
- 动作类型
- 是否成功
- 结果类型
- 错误信息
- 执行耗时
- 点击次数
- 移动次数
- 候选点击点序列
- 最后一次点击
- 对话文本摘要

## 当前总结方式

1. 先对样本做确定性统计：
   - 成功率
   - 平均点击次数
   - 平均移动次数
   - 平均耗时
   - 候选点击点表现
   - 常见失败原因
2. 再调用本地文本模型，把统计结果翻译成短总结。

## 后续升级方向

- 把高成功率点击点排序固化到点击链路
- 把高频失败模式转成更明确的场景退出条件
- 对不同场景维护不同点击点集合
- 引入交互前后截图样本，用于后续模板匹配校准
