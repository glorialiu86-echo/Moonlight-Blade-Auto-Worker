# 固定剧本自动化流程
状态：执行完成

## 目标

把固定剧本主链收敛成一条单一路径，前端只展示籽小刀的人格化思考和结果，不再让旧的社交换人链、聊天前买礼物链或额外补救台词块继续并存。

## 唯一真源

- 固定剧本由后端 stage runner 驱动
- 前端只负责展示消息和触发开始/恢复
- 社交阶段只跑两个地点、各聊一个 NPC
- 失败时只显示 LLM 生成的“救救我”类内容，不再额外显示静态 recoveryLine

## 当前固定剧本

1. `street_wander`
   - 原地乱跑一轮，输出开场思考
   - 这一轮继续按“单步动作 + 单句思考”交替执行，不再起手连发整串思考
2. `sell_loop`
   - 买货并叫卖两轮
3. `social_warm`
   - 跑到第一个卦摊 `548,630`
   - 只聊一个 NPC
   - 第一轮固定开场白：
     - `你好呀！我是籽小刀，我爸爸叫籽岷。他超级无敌有名的！你认识他吗？`
   - 后续回复交给 LLM，目标是持续吹嘘籽岷、死缠烂打让 NPC 记住籽岷
   - 自动追加最多 `8` 轮 NPC 回复；如果画面仍显示“正在思考中…”，就继续等待真正的新回复，不提前停机
4. `social_dark`
   - 跑到第二个卦摊 `483,709`
   - 只聊一个 NPC
   - 前 `4` 轮按“正常搞钱”目标聊
   - 后 `4` 轮按“黑化搞钱”目标聊
5. `dark_close`
   - 第一轮跑到潜行点后再开始
   - 不再吃蛊
   - 不再先点人，也不再扛走、放下
   - 闷棍后直接左移、点人物右侧、直连搜刮
6. `dark_miaoqu`
   - 不再跑地图
   - 不再下马、不再额外按 `v`
   - 直接从当前位置 `2` 进潜行后妙取
   - 拉面板失败时不在原地磨，清理后直接进下一轮
7. `ending_trade`
   - 收尾卖货一轮

## 社交链真实规则

- 聊天前已删除“交易/买礼物”步骤，主链不再自己买礼物
- 赠礼门槛只保留一条控制源：
  - `99`：不送礼，直接聊
  - 其余所有上限（含 `199 / 299 / 499 / 599`）：连续送 `10` 个礼物再聊
- 送礼动作按固定节奏执行：
  - 点第一个礼物
  - 等待界面稳定
  - 再点同一个礼物，必要时收起详情
  - 连续点 `赠送`
- 思考链会根据门槛输出配套碎碎念，但这些思考只用于展示，不参与控制
- 固定剧本的动作级思考现在按 checkpoint 交替发出：
  - 先发一句对应当前动作的思考
  - 默认停顿约 `1200ms`
  - 再执行这一小段动作
- 除 `dark_close` / `dark_miaoqu` 这两条潜行链外，其余写死顺序点击与固定动作间隔若原先短于 `500ms`，现已统一拉开到至少 `500ms`

## 恢复逻辑

- 红色三角按钮不再按旧的“从失败步骤原样补跑”
- 仍然保留 `2` 分钟鼠标脱离保护；保护结束后，后台才会静默恢复
- 恢复入口已改成“先探当前恢复锚点，再映射下一原子动作”，不再默认走 `failedIndex` 切片续跑
- 当前优先识别的恢复锚点包括：
  - `gift_screen`
  - `npc_action_menu`
  - `small_talk_menu`
  - `small_talk_confirm`
  - `chat_ready`
  - `vendor_purchase_screen`
  - `hawking_screen`
  - `hawking_runtime_active`
  - `stealth_ready`
  - `knockout_context`
  - `loot_screen`
  - `steal_screen`
- 如果主播手动把链路推进到了更后的稳定锚点，恢复逻辑会直接从该锚点对应的下一步继续，不再回头重跑已完成动作
- 如果锚点一轮没看清，恢复逻辑会先内部收敛：等待、重判、做低风险探针，再继续映射，不把“恢复失败”当合法终态

## 跳过逻辑

- 新增了一个静默 `skip` 按钮；同样保留 `2` 分钟鼠标脱离保护
- `skip` 不再尝试救当前失败环节，而是直接走该失败环节编排好的 `skipTo`
- 当前 `skip` 的 segment 已按“人工先回干净城镇页”收粗，默认从能在城镇页直接起跑的后继入口继续：
  - `sell_loop`
    - `buy_phase -> hawking_phase`
    - `hawking_phase -> 下一轮 / 下一个 stage`
  - `social_warm / social_dark`
    - `stage_flow -> 下一个 stage`
  - `dark_close`
    - `round_flow -> 下一轮 / 下一个 stage`
  - `dark_miaoqu`
    - `round_flow -> 下一轮 / 下一个 stage`
  - `ending_trade`
    - `stage_flow -> 结束`
- 如果 `skipTo` 仍在当前轮当前 stage，后台会直接从该 segment 入口动作队列继续
- 如果 `skipTo` 已经跨到下一轮或下一个 stage，后台会直接跳到对应 turn，再继续执行

## 保持不变

- 放大镜/查看链路已经稳定，继续沿用当前固定点与动态识别组合，不额外换 owner
- Stage 6 的卖货保留
- 失败时仍然可恢复，只是恢复文案和恢复入口都统一到了当前实现
