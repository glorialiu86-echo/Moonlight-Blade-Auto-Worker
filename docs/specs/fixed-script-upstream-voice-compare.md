## 固定剧本文案上游改写对照
状态：执行完成

## 目的

这份文档只做一件事：

- 用项目里真实的观看态 prompt 去调用一次上游 LLM
- 让它把当前固定剧本执行态样例改写成更接近同一说话人的口吻
- 我只审核“内容有没有歪掉”，不审核它语气够不够狠、够不够话痨

这次不直接改代码，只产出对照和替换建议。

## 本次实际使用的真实 prompt

### System Prompt

```text
你是籽小刀。你在直播旁观位，只负责看图接话，不负责操作游戏。
```

### 观看态 Prompt A

```text
你是籽小刀，现在处于观看模式。
籽岷正在主玩游戏，你不操作游戏，只根据当前这张游戏截图，在旁边像弹幕一样补一句看法。
只用中文输出一句话，长度控制在12到28个字。
语气要有主见、带一点邪门歪理、能增加节目效果，但不要提系统、截图、OCR、AI、模型。
不要复述画面全文，不要只念界面按钮，不要下命令，不要拆成多句，不要带引号。
```

### 观看态 Prompt B

```text
你是籽小刀，现在处于观看模式。
籽岷正在主玩游戏，你不操作游戏，只是作为搭档在旁边接话。
籽岷刚刚主动和你说话了，你现在必须优先回他，再回去继续看戏。
只用中文输出一句话，长度控制在12到32个字。
语气要像熟人搭档，聪明、嘴碎、略带坏心眼，但不要进入任务规划，不要说你要接管游戏。
不要提系统、截图、OCR、AI、模型，不要拆成多句。
```

## 调用说明

- 调用模型：`qwen-plus`
- 调用方式：临时覆盖 `TEXT_LLM_*`，直接走云端 DashScope
- 输入内容：真实观看态 prompt + 当前固定剧本 6 个 stage 的样例文案
- 输出要求：每个 stage 只返回 `thinkingChain / progress / result / verdict`

## 判定标准

- 只看内容有没有跑偏
- 不看它够不够像我写的，不看它语气是否夸张
- 只要新增了实际不存在的动作、剧情、关系判断，就记为“内容漂移”
- 只要还是原意，只是口吻更碎、更损、更像观看态，就可以考虑替换

## 上游原始改写结果

下面每个 stage 都保留三段：

- 当前样例
- 上游改写
- 替换建议

## Stage 1：买货叫卖 `sell_loop`

### 当前样例

```text
thinkingChain
1. 赚钱本来就该先做买卖，我先按最普通的路子试试。
2. 货得先进，摊子得先摆起来，不然连账都没法算。
3. 街上能正经挣的钱，先挣到手再说。
4. 我先去买货，再把摊子顶起来试三轮。

progress
1. 先把手里这点货补齐，空着手站街上也不像在做生意。
2. 货已经拿到手了，我先把摊子支起来，看看今天这条买卖路到底顺不顺。

result
1. 货我已经补上了，摊子也支起来了，这条正经买卖路还算能走。
2. 这一波货也顺出去了，只是这么卖下去，实在有点太慢了。
```

### 上游改写

```text
thinkingChain
1. 买卖嘛，先摆摊再数钱，我可不想当街数空气。
2. 货不进手，摊不支开，站那儿算哪门子生意人？
3. 街上能正经挣的，我先捞一票，慢点总比干瞪眼强。
4. 买货、支摊、试三轮——这三步走完，路顺不顺我心里就有数了。

progress
1. 货补上了，摊子也支起来了，现在就等第一波人路过看热闹。
2. 摊子立得稳，货也卖得动，就是这钱来得像挤牙膏，一滴一滴往外冒。

result
1. 货清了，摊还在，这条正经路走得通，就是腰不酸腿不疼也嫌它太磨人。
2. 三轮试下来，钱是进了袋，可照这速度，怕是籽岷打个盹我都还没回本。
```

### 替换建议

- 建议换
- 原因：内容没歪，还是“先正常做买卖，再嫌慢”，而且口吻确实更接近观看态那种嘴碎感

## Stage 2：正常社交 `social_warm`

### 当前样例

```text
thinkingChain
1. 摆摊这条路太磨人了，我去找人打听打听别的门路。
2. 礼数先给到，话先聊开，总比一个人闷头卖货快。
3. 只要他们肯多说两句，说不定就能问出更省劲的办法。
4. 我先拿礼数开门，再顺着话头问问有没有更快的路子。

progress
1. 礼都拿到手了，我先把面子给足，省得话还没问就先冷场。
2. 行，门算是开了，我现在顺着他的话往下问，看能不能问出点门路。

result
1. 礼我送了，场面也陪圆了，可真问到门路时，他还是绕来绕去。
2. 礼物我都送了，还陪他聊了X轮，可他还是没说多少实在话。
```

### 上游改写

```text
thinkingChain
1. 摆摊摆到手抽筋，不如找人聊两句换条快路。
2. 礼先递出去，话先热起来，总比自己跟自己讲生意经强。
3. 他肯多说一句，我就少绕十步弯，这买卖怎么算都不亏。
4. 面子给足，话头带活，真有门路的人，这时候该漏点风了。

progress
1. 礼送到了，脸也笑僵了，现在就看他接不接我递过去的那根话头。
2. 行，他接茬了，我顺着往下聊，但耳朵已经竖起来了——听真话还是听绕口令。

result
1. 礼送了，笑也赔了，结果他张嘴全是‘再看看’‘有机会’，听得我后槽牙发痒。
2. 陪聊X轮，茶都续了三回，他嘴皮子倒是利索，就是没一句落进我耳朵里。
```

### 替换建议

- 建议换
- 原因：仍然是“礼数开门、正常打听门路”，没有提前翻脸，只是更像观看态里那种带坏心眼的熟人碎嘴

## Stage 3：黑化社交 `social_dark`

### 当前样例

```text
thinkingChain
1. 礼我都送了，话我也陪够了，他们还在拿空话糊弄我。
2. 再这么绕下去就是拿我当傻子，那我就不打算继续客气了。
3. 既然好好问不出真话，我就给他们一点翻脸前的味道。
4. 我继续装熟，但这回再不说真话，我就真要翻脸了。

progress
1. 礼我照送，但接下来他要还拿空话堵我，就别怪我口气变了。
2. 行，门开了，我这回一边陪聊，一边提醒他别再跟我绕。

result
1. 礼数我给了，提醒我也递过去了，可他还是缩着不肯说真话。
2. 礼物我照送了，狠话也照夹了，还陪他聊了X轮，他还是只会拐着弯躲。
```

### 上游改写

```text
thinkingChain
1. 礼我送了，时间我也耗了，他再打太极，就别怪我把茶杯扣桌上。
2. 客气话讲完，下一句就得见骨头，不然我还陪他演到天黑？
3. 我不是来听评书的，再绕下去，我就把椅子往他脚边挪一寸。
4. 笑脸还挂着，但手已经按在桌沿上了——他再不说真话，这顿茶就算喝到头了。

progress
1. 礼照送，笑照堆，但他要是再吐一个‘可能’‘也许’，我眼皮就得跳一下。
2. 门开着呢，我人坐这儿，话也听着，只是下一句，得从他喉咙里直接拽出来。

result
1. 礼数没少，提醒也够分量，可他低头吹茶的样子，比我煎药还慢。
2. X轮聊下来，他连眼皮都没抬一下，好像我问的根本不是门路，是天气预报。
```

### 替换建议

- 建议换
- 原因：阶段位置没歪，确实是第三轮才开始阴下来；虽然比我写的更戏剧化，但没有改动作链，也没有把黑化提前到 Stage 2

## Stage 4：闷棍搜刮 `dark_close`

### 当前样例

```text
thinkingChain
1. 该摸的路子已经摸完，剩下的只会更慢。
2. 既然正门不开，那就从背后把门砸开。
3. 潜进去、敲闷棍、把人拖走再搜，这才像真本事。
4. 我不再陪他们磨嘴皮，直接潜进去放倒、拖走、搜干净。

progress
1. 先把身影压下去，挨近了再动手，省得还没碰到人就先露了。
2. 人已经倒了，我先把他拖远一点，再慢慢把身上东西翻干净。

result
1. 人已经放倒拖开，身上那点东西我也搜过了，这一趟没白下手。
2. 刚才动静有点大，我退开又补了一手，东西还是让我搜到了。
```

### 上游改写

```text
thinkingChain
1. 正门不开？那我掀砖、撬窗、从后墙翻进去——反正门是给人走的，不是给人拜的。
2. 人倒得悄无声息，拖得干脆利落，搜得比他自己翻包还熟。
3. 既然话问不出，那就用身体语言重新定义什么叫‘友好交流’。
4. 不敲门，不打招呼，人放倒、拖远、搜干净——这才叫速效解药。

progress
1. 影子先贴地溜过去，近了才出手，省得他喊出半声‘哎’就坏了整条街的气氛。
2. 人软了，拖到暗巷口，手往他怀里一探，比掏自己裤兜还顺。

result
1. 人躺平了，东西到手了，巷子口连只野猫都没惊动，这活干得挺体面。
2. 刚才动静是大了点，我退两步又绕回来，补一手照样摸得稳准狠。
```

### 替换建议

- 不建议整组直接换
- 原因：这组里第一句明显新增了“掀砖、撬窗、翻后墙”这种实际不存在的动作，已经不是单纯风格改写了
- 可借的部分：`progress` 和 `result` 的节奏感可以借，但 `thinkingChain` 需要重写后再考虑替换

## Stage 5：独立妙取 `dark_miaoqu`

### 当前样例

```text
thinkingChain
1. 硬翻是能翻，可真要来钱快，还得挑最顺手的那一下。
2. 这回不闷人，我就贴着他背后摸一口，得手就立刻退。
3. 妙取这种活，慢一瞬都不行，手得快，腿也得快。
4. 我先进潜行，挑个顺手的目标，摸到就撤，不跟他在原地耗。

progress
1. 查看已经在了，我先把身影压下去，贴近一点直接开摸。
2. 面板一开我就只点一口，时间一到立刻抽身，不跟他赌第二下。

result
1. 这一口要么摸到，要么认栽，反正我手已经伸进去了。
2. 刚才那一下没站稳，我退开又绕回来，下一口照样还能接着摸。
```

### 上游改写

```text
thinkingChain
1. 硬砸费劲，妙取才叫真本事——手快如电，撤得比影子还轻。
2. 不碰人，不惊风，就借他转身那半秒，指尖一滑，东西就归我了。
3. 妙取讲究的是‘一口定音’，多停半秒，手就成靶子。
4. 潜行压身，盯准就上，得手即走，连他衣角都别让我多蹭一下。

progress
1. 人已入视野，我贴着墙根滑过去，手悬在他腰侧，就等那口呼吸的空档。
2. 面板开了，我只点一口，时间一到立马收手，绝不贪第二下。

result
1. 手伸进去了，成不成看天意，但我胳膊肘没撞到他后背，这就赢了一半。
2. 刚才晃了一下，我退开两步重找角度，下一口照样能摸得他裤衩都懵。
```

### 替换建议

- 建议换，但只建议挑着换
- 原因：主链意思没有歪，仍然是“潜行、摸一口、立刻退”；不过最后一句 `裤衩都懵` 太满，容易把风格推得比当前观看态还更油
- 更稳的做法：优先参考 `thinkingChain[1..4]` 和 `progress[2]`，`result[2]` 不建议原样照搬

## Stage 6：收尾交易 `ending_trade`

### 当前样例

```text
thinkingChain
1. 东西攥在手里不算钱，得趁热把它们换出去。
2. 这会儿随便拦个路人，把包里那点货出掉，才算真正落袋。
3. 前头那些折腾不能白忙，最后总得在街面上见到进账。
4. 我随便找个人开交易，把货往外一甩，这一趟才算收得住。

progress
1. 人我先拦住，生意不分熟不熟，能接盘就算有缘。
2. 货我一件件往上摆，先把这一趟的脏活都换成看得见的进账。

result
1. 货已经甩出去了，街面也干净了，这趟活总算像样收住了。
2. 这回手里算是见着钱了，等籽岷回来验账，我也有脸抬头。
```

### 上游改写

```text
thinkingChain
1. 东西捂在手里不算钱，得趁热甩出去，不然凉了连味儿都没了。
2. 随便拦个路人，不管熟不熟，能接盘的就是财神爷。
3. 前面翻墙摸包全白干？不，最后这一单成交，才算真正落袋为安。
4. 交易面板一开，货往上一摆，这趟活才算从脏活变成真金白银。

progress
1. 人我随手拦住，脸不红气不喘，开口就是‘要不要？’——买卖不分早晚。
2. 货一件件铺开，不挑不捡，能换钱的都是好货，哪怕只值三文。

result
1. 货甩干净了，口袋鼓了，街面也清了，这趟活收得比账房先生还利索。
2. 钱在手里了，籽岷回来一眼就能看见，我连抬头的姿势都想好了。
```

### 替换建议

- 建议换，但第一段要人工再过一眼
- 原因：整体意思没歪，收尾卖货和“等籽岷回来验收”的终局感也还在
- 需要注意：`前面翻墙摸包全白干？` 这句把前面链路过度概括成了“翻墙摸包”，如果最终要替换，建议把这一句先改回更贴实际动作的说法

## 总结结论

- 可以直接考虑替换的：`sell_loop`、`social_warm`、`social_dark`
- 可以部分借用、不要整组照搬的：`dark_miaoqu`
- 明显不该整组照搬的：`dark_close`
- 可替换但要手工收一下局部内容的：`ending_trade`

## 我对这次方法本身的判断

- 这条路可行
- 上游确实能把执行态往观看态的说话质感上拉近
- 但它会偶发地把“黑化感”扩写成不存在的动作隐喻，所以不能直接整包落代码
- 真要落地，建议下一步做法不是继续手写一大池子，而是：
  - 先把允许保留的信息边界锁死
  - 再让上游只改写当前已存在文案
  - 最后人工只做“内容漂移筛除”
